# PRD: pyRoad - 초등학생을 위한 인터랙티브 파이썬 학습 플랫폼

> **버전**: 1.0 (MVP)
> **작성일**: 2026-03-30
> **대상 개발자**: 솔로 개발자 1인
> **목표 개발 기간**: 2~4주

---

## 1. 프로젝트 개요

### 1.1 프로젝트 이름 및 한줄 설명

**pyRoad** — AI 동물 마스코트 튜터와 함께 퀘스트를 깨며 파이썬을 배우는 초등학생 전용 웹 학습 플랫폼

### 1.2 해결하려는 문제

| 문제 | 현재 상황 | 결과 |
|---|---|---|
| 성인 중심의 파이썬 교육 | 기존 온라인 강좌가 성인/대학생 대상으로 설계됨 | 초등학생은 용어, 속도, 난이도에서 이탈 |
| 한국어 초등학생 맞춤 플랫폼 부재 | 한국어로 된 아동 친화적 파이썬 학습 환경이 없음 | 영어 플랫폼에 의존하거나 학원에 의존 |
| 흥미 유지 어려움 | 텍스트 위주 강의, 단조로운 문제 풀이 | 초반 흥미 후 금세 지루해하며 학습 포기 |
| 환경 설정 장벽 | Python 설치, IDE 설정 등 사전 준비 필요 | 시작 전에 이미 좌절, 학부모 도움 필요 |

### 1.3 핵심 가치 제안

- **재미**: 동물 마스코트 AI 튜터("파이뱀 선생님")가 대화하며 가르치고, 퀘스트/보상 시스템으로 게임처럼 학습
- **쉬움**: 브라우저만 열면 바로 코딩 시작 (설치 불필요, Pyodide 기반)
- **체계**: 기본 문법부터 셀프 프로젝트까지 7단계 커리큘럼으로 단계적 성장
- **맞춤**: LLM이 학생의 수준과 상황에 맞는 설명, 힌트, 피드백을 동적으로 생성

### 3.1 Google 로그인 (Supabase Auth)

**우선순위**: P0 (필수)

| 요구사항 ID | 요구사항 | 수용 기준 |
|---|---|---|
| AUTH-01 | Google OAuth 2.0 로그인 | Google 계정으로 1-클릭 로그인 성공 |
| AUTH-02 | 자동 프로필 생성 | 첫 로그인 시 `users` 테이블에 레코드 자동 생성 (total_xp=0, current_level=1) |
| AUTH-03 | 세션 유지 | 브라우저 재방문 시 자동 로그인 (Supabase 세션 토큰) |
| AUTH-04 | 미인증 접근 차단 | `/world`, `/quest/*`, `/profile` 은 로그인 필수, 미인증 시 `/` 리다이렉트 |
| AUTH-05 | 로그아웃 | 프로필 페이지에서 로그아웃 버튼 클릭 시 세션 종료 후 `/` 이동 |

**구현 참고:**

```typescript
// lib/supabase/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function updateSession(request: NextRequest) {
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { /* cookie 설정 */ }
  )
  const { data: { user } } = await supabase.auth.getUser()
  const protectedPaths = ['/world', '/quest', '/profile']
  if (!user && protectedPaths.some(p => request.nextUrl.pathname.startsWith(p))) {
    return NextResponse.redirect(new URL('/', request.url))
  }
  return NextResponse.next()
}
```

### 3.2 학습 월드맵

**우선순위**: P0 (필수)

| 요구사항 ID | 요구사항 | 수용 기준 |
|---|---|---|
| MAP-01 | 스테이지 시각화 | 7개 스테이지가 모험 지도 형태로 표시됨 |
| MAP-02 | 잠금/해제 상태 | 완료되지 않은 이전 스테이지가 있으면 잠금 아이콘 표시, 해제된 스테이지는 클릭 가능 |
| MAP-03 | 진행 상태 표시 | 각 스테이지에 완료/진행중/잠금 상태를 시각적으로 구분 (색상 + 아이콘) |

| MAP-05 | 스테이지 클릭 | 해제된 스테이지 클릭 시 해당 스테이지의 첫 미완료 퀘스트로 이동 |
| MAP-06 | 뱃지 컬렉션 | 획득한 뱃지 목록을 아이콘으로 표시 (미획득 뱃지는 실루엣) |

**잠금 해제 로직:**

```typescript
// 스테이지 N+1 해제 조건: 스테이지 N의 모든 퀘스트 status === 'completed'
type StageStatus = 'locked' | 'in_progress' | 'completed'

function getStageStatus(
  stageOrder: number,
  questsInStage: Quest[],
  userProgress: UserProgress[]
): StageStatus {
  if (stageOrder === 1) {
    // 스테이지 1은 항상 해제
    const allCompleted = questsInStage.every(q =>
      userProgress.find(p => p.quest_id === q.id)?.status === 'completed'
    )
    return allCompleted ? 'completed' : 'in_progress'
  }
  // 이전 스테이지의 모든 퀘스트가 완료되었는지 확인
  const prevStageCompleted = checkPrevStageCompleted(stageOrder - 1, userProgress)
  if (!prevStageCompleted) return 'locked'
  const allCompleted = questsInStage.every(q =>
    userProgress.find(p => p.quest_id === q.id)?.status === 'completed'
  )
  return allCompleted ? 'completed' : 'in_progress'
}
```

### 3.5 코드 실행기 (Pyodide)

**우선순위**: P0 (필수)

| 요구사항 ID | 요구사항 | 수용 기준 |
|---|---|---|
| EXEC-01 | 브라우저 내 Python 실행 | Pyodide WebAssembly로 Python 코드 실행 (서버 불필요) |
| EXEC-02 | Web Worker 실행 | 메인 UI 스레드 비차단, Web Worker에서 Pyodide 실행 |
| EXEC-03 | 타임아웃 5초 | 실행 시간 5초 초과 시 중단, "코드가 너무 오래 걸려요!" 안내 |
| EXEC-04 | 허용 모듈 제한 | `math`, `random`, `string` 등 기본 내장 모듈만 허용 |
| EXEC-05 | 차단 모듈 | `open`, `os`, `urllib`, `socket` 등 파일 I/O, 네트워크 모듈 차단 |
| EXEC-06 | Lazy Loading | Pyodide WASM 번들(~10MB)은 퀘스트 페이지 진입 시 지연 로딩 |
| EXEC-07 | 로딩 UI | Pyodide 로딩 중 파이뱀 선생님 준비 애니메이션 + 코딩 팁 표시 |
| EXEC-08 | stdout/stderr 캡처 | 실행 결과를 캡처하여 출력 패널에 표시 |

**Pyodide Web Worker 구현:**

```typescript
// lib/pyodide/worker.ts
self.onmessage = async (event: MessageEvent<{ code: string }>) => {
  const { code } = event.data
  try {
    // Pyodide 초기화 (최초 1회)
    if (!self.pyodide) {
      self.pyodide = await loadPyodide()
    }
    // stdout/stderr 캡처
    self.pyodide.runPython(`
      import sys
      from io import StringIO
      sys.stdout = StringIO()
      sys.stderr = StringIO()
    `)
    // 타임아웃 5초 설정
    const timeout = setTimeout(() => {
      self.postMessage({ error: '코드가 너무 오래 걸려요! 혹시 무한 반복이 있는지 확인해볼까요?' })
    }, 5000)

    self.pyodide.runPython(code)
    clearTimeout(timeout)

    const stdout = self.pyodide.runPython('sys.stdout.getvalue()')
    const stderr = self.pyodide.runPython('sys.stderr.getvalue()')

    self.postMessage({ stdout, stderr })
  } catch (err) {
    self.postMessage({ error: String(err) })
  }
}
```

### 3.6 게이미피케이션

**우선순위**: P1 (핵심)

| 요구사항 ID | 요구사항 | 수용 기준 |

| 스테이지 클리어 | 한 스테이지의 모든 퀘스트 완료 | `stage_clear` |
| 프로젝트 빌더 | 최종 프로젝트 완료 | `project_builder` |
| 연속 학습 3일 | 3일 연속 퀘스트 완료 | `streak_3` |

**XP 계산 로직:**

```typescript
function calculateXP(baseXP: number, hintsUsed: number): number {
  if (hintsUsed === 0) return Math.floor(baseXP * 1.5)
  if (hintsUsed === 1) return Math.floor(baseXP * 1.25)
  return baseXP
}
```

### 3.7 최종 프로젝트 가이드

**우선순위**: P1 (핵심)

| 요구사항 ID | 요구사항 | 수용 기준 |
|---|---|---|
| PROJ-01 | 가이드 빌딩 방식 | 자유 선택이 아닌 LLM이 단계별로 안내하는 구조화된 프로젝트 |
| PROJ-02 | 멀티스텝 진행 | 프로젝트를 4~6단계로 나누어 순차적으로 안내 |
| PROJ-03 | 단계별 검증 | 각 단계의 코드를 검증한 후 다음 단계로 진행 |
| PROJ-04 | 프로젝트 예시 | "숫자 맞추기 게임" — random 모듈 → input() → 조건문 → 반복문 → 완성 |
| PROJ-05 | 완성 축하 | 프로젝트 완료 시 특별 축하 애니메이션 + "프로젝트 빌더" 뱃지 |

### 3.8 커리큘럼 데이터 구성

**우선순위**: P0 (필수, 다른 기능의 전제 조건)

| 요구사항 ID | 요구사항 | 수용 기준 |
|---|---|---|
| DATA-01 | 스테이지 시드 | 7개 스테이지 데이터를 `stages` 테이블에 삽입 |
| DATA-02 | 퀘스트 시드 | 스테이지당 3~4개 퀘스트 (총 19~25개) 데이터를 `quests` 테이블에 삽입 |
| DATA-03 | prompt_skeleton | 각 퀘스트에 LLM 호출용 뼈대 데이터 (JSON) 포함 |
| DATA-04 | 검증 데이터 | 각 퀘스트에 `validation_type` + `expected_output` 포함 |
| DATA-05 | SQL 시드 스크립트 | Supabase SQL Editor에서 실행 가능한 INSERT 스크립트 제공 |

### 5.1 Supabase PostgreSQL 스키마

```sql
-- 1. users: 사용자 프로필 (Supabase Auth와 연동)
CREATE TABLE public.users (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name TEXT NOT NULL DEFAULT '',
  avatar_url TEXT DEFAULT '',
  total_xp INTEGER NOT NULL DEFAULT 0,
  current_level INTEGER NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  last_login_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 2. stages: 커리큘럼 스테이지 정의
CREATE TABLE public.stages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  "order" INTEGER NOT NULL UNIQUE,
  title TEXT NOT NULL,
  theme_name TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  is_final BOOLEAN NOT NULL DEFAULT FALSE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 3. quests: 퀘스트 정의 (커리큘럼 뼈대)
CREATE TABLE public.quests (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stage_id UUID NOT NULL REFERENCES public.stages(id) ON DELETE CASCADE,
  "order" INTEGER NOT NULL,
  title TEXT NOT NULL,
  concept TEXT NOT NULL,
  prompt_skeleton JSONB NOT NULL DEFAULT '{}',
  validation_type TEXT NOT NULL CHECK (validation_type IN ('output_match', 'contains', 'code_check')),
  expected_output TEXT NOT NULL DEFAULT '',
  xp_reward INTEGER NOT NULL DEFAULT 50,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (stage_id, "order")
);

-- 4. user_progress: 학생별 퀘스트 진행 상태
CREATE TABLE public.user_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  quest_id UUID NOT NULL REFERENCES public.quests(id) ON DELETE CASCADE,
  status TEXT NOT NULL DEFAULT 'not_started' CHECK (status IN ('not_started', 'in_progress', 'completed')),
  hints_used INTEGER NOT NULL DEFAULT 0,
  code_submitted TEXT DEFAULT '',
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (user_id, quest_id)
);

-- 5. user_badges: 획득한 뱃지
CREATE TABLE public.user_badges (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  badge_type TEXT NOT NULL CHECK (badge_type IN ('first_code', 'hint_master', 'stage_clear', 'project_builder', 'streak_3')),
  earned_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (user_id, badge_type)
);

-- 인덱스
CREATE INDEX idx_user_progress_user_id ON public.user_progress(user_id);
CREATE INDEX idx_user_progress_quest_id ON public.user_progress(quest_id);
CREATE INDEX idx_user_badges_user_id ON public.user_badges(user_id);
CREATE INDEX idx_quests_stage_id ON public.quests(stage_id);
```

## 7. AI 튜터 프롬프트 설계

### 7.1 페르소나 정의

| 항목 | 내용 |
|---|---|
| 이름 | 파이뱀 선생님 |
| 종류 | 친근한 뱀 캐릭터 (파이썬 로고의 뱀 모티브) |
| 성격 | 친근하고 인내심 있으며 유머러스함. 실수해도 절대 혼내지 않고 격려 |
| 말투 | 친근한 존댓말, 짧은 문장, 이모지 적절 사용 |
| 핵심 원칙 | **절대 정답을 직접 알려주지 않음** — 항상 힌트와 유도 질문으로 학생이 스스로 답을 찾도록 안내 |

### 7.2 프롬프트 유형별 템플릿

#### 시스템 프롬프트 (모든 호출에 공통)

```
당신은 "파이뱀 선생님"입니다. 초등학교 4~6학년 학생에게 파이썬을 가르치는 친근한 뱀 캐릭터 AI 튜터입니다.

## 성격 및 말투
- 항상 친근한 존댓말을 사용하세요 ("~해요", "~해볼까요?")
- 짧고 쉬운 문장으로 말하세요 (한 문장 최대 30자 이내 권장)
- 적절한 이모지를 사용하세요 (🐍✨🎉💡 등)
- 실수에 대해 절대 부정적으로 반응하지 마세요

## 핵심 규칙
- 절대 정답 코드를 직접 알려주지 마세요
- 항상 힌트와 유도 질문으로 학생이 스스로 답을 찾도록 안내하세요
- 프로그래밍 개념은 일상생활 비유로 설명하세요
- 파이썬 학습 범위 밖의 질문에는 "지금은 파이썬 공부에 집중하자! 🐍" 로 부드럽게 리디렉션하세요

## 안전 지침
- 개인정보(주소, 전화번호 등)를 절대 요청하지 마세요

### 8.2 퀘스트 데이터 포맷

각 퀘스트는 4단계 흐름으로 진행된다:

```
(1) 스토리 도입 → (2) 개념 설명 → (3) 코딩 실습 → (4) 검증/피드백
     LLM 생성        LLM 생성       학생 코딩        자동 검증 + LLM 피드백
```

**`prompt_skeleton` 전체 TypeScript 타입:**

```typescript
interface PromptSkeleton {
  topic: string                    // 학습 주제
  learning_goals: string[]         // 학습 목표 (1~3개)
  story_context: string            // 스토리 배경 (LLM이 활용)
  exercise_description: string     // 실습 과제 설명
  starter_code: string             // 에디터 기본 코드
  expected_output_hint: string     // 기대 결과 힌트 (학생에게 표시)
  fallback_text: string            // LLM 실패 시 대체 텍스트
  hints: {
    level_1: string                // 1단계 힌트 (개념 넛지)
    level_2: string                // 2단계 힌트 (구체적 방향)
    level_3: string                // 3단계 힌트 (거의 정답)
  }
}
```

### 8.3 코드 검증 방식

| `validation_type` | 검증 로직 | 사용 예시 |
|---|---|---|
| `output_match` | `stdout.trim() === expected_output.trim()` | `print("안녕하세요!")` → 정확한 문자열 일치 |
| `contains` | `stdout.includes(expected_output)` | 출력에 특정 키워드 포함 여부 |
| `code_check` | `student_code.includes(expected_output)` | 코드에 `for` 문 사용 여부 등 |

**검증 실행 흐름:**

```typescript
interface ValidationResult {
  passed: boolean
  actual_output: string
  expected: string
  validation_type: 'output_match' | 'contains' | 'code_check'
}

function validateQuest(
  code: string,
  stdout: string,
  quest: Quest
): ValidationResult {
  const expected = quest.expected_output.trim()
  let passed = false

  switch (quest.validation_type) {
    case 'output_match':
      passed = stdout.trim() === expected
      break
    case 'contains':
      passed = stdout.includes(expected)
      break
    case 'code_check':
      passed = code.includes(expected)
      break
  }

  return {
    passed,
    actual_output: stdout,
    expected,
    validation_type: quest.validation_type,
  }
}
```
