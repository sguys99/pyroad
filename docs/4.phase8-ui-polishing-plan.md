## 작업 목록 (8개 Task)

### Task 8-1: 클릭 타겟 최소 44px 보정

**문제**: Button 컴포넌트의 기본 높이가 h-8(32px)으로 아동 UX 기준 44px 미달.

**수정 대상 파일**:
- `src/components/ui/button.tsx` — size variant 조정
- `src/components/quest/ConversationPanel.tsx` — 힌트 버튼 size
- `src/components/quest/QuestShell.tsx` — 모바일 탭 바 높이
- `src/components/quest/ProjectQuestShell.tsx` — 모바일 탭 바 높이
- `src/components/quest/QuestHeader.tsx` — 뒤로가기 링크 터치 영역

**구체적 변경**:
1. `button.tsx`: `default` size를 `h-11`(44px)로 변경, `sm`을 `h-9`(36px)->`h-10`(40px), `lg`를 `h-12`(48px)로 변경
2. 모바일 탭 바: `py-2.5` -> `py-3` (총 높이 ~44px 이상 확보)
3. QuestHeader "월드맵" 링크: `min-h-[44px] flex items-center` 적용하여 터치 영역 확보

### Acceptance criteria
- [ ] 모든 버튼의 렌더링 높이가 44px 이상이다
- [ ] 모바일 탭 바의 각 탭 터치 영역이 44px 이상이다
- [ ] QuestHeader 뒤로가기 링크의 터치 영역이 44px 이상이다

### Task 8-2: 버튼 Framer Motion scale/spring 효과

**문제**: StageNode와 축하 팝업 버튼에만 적용됨. 나머지 주요 인터랙션 버튼에 미적용.

**수정 대상 파일**:
- `src/components/LoginButton.tsx` — motion.div 래핑
- `src/components/LogoutButton.tsx` — motion.div 래핑
- `src/components/quest/ConversationPanel.tsx` — 힌트 버튼 애니메이션
- `src/components/quest/CodePanel.tsx` — "코드 돌려보기!" 버튼에 whileHover 추가

**구체적 변경**:
- 모든 주요 CTA 버튼에 `whileHover={{ scale: 1.03 }}` + `whileTap={{ scale: 0.97 }}` + `transition={{ type: 'spring', stiffness: 400, damping: 17 }}` 적용
- `motion.div` 래퍼 패턴 사용

### Acceptance criteria
- [ ] LoginButton 호버/클릭 시 scale 애니메이션이 동작한다
- [ ] LogoutButton 호버/클릭 시 scale 애니메이션이 동작한다
- [ ] 힌트 받기 버튼 호버/클릭 시 scale 애니메이션이 동작한다
- [ ] "코드 돌려보기!" 버튼 호버/클릭 시 scale 애니메이션이 동작한다

### Task 8-3: 에러 메시지를 파이뱀 선생님 안내로 교체

**문제**: OutputPanel의 에러 표시가 `AlertTriangle` + `destructive` 색상 -> PRD의 "빨간 경고 절대 사용 금지" 위반.

**수정 대상 파일**:
- `src/components/quest/OutputPanel.tsx`
- `src/components/quest/CodePanel.tsx` (Pyodide 에러 메시지)

**구체적 변경**:
1. OutputPanel 에러 표시: `AlertTriangle` 아이콘 -> 파이뱀 선생님 아바타, `destructive` 배경색 -> `accent/10` 부드러운 톤
2. "앗, 오류가 있어요!" -> "앗, 여기를 한번 살펴볼까요?" 스타일
3. 타임아웃 메시지: 파이뱀 선생님 아이콘 적용
4. CodePanel의 Pyodide 에러: `text-destructive` -> 파이뱀 선생님 스타일 안내

### Acceptance criteria
- [ ] 코드 실행 에러 시 빨간 경고 아이콘 대신 파이뱀 선생님 아바타가 표시된다
- [ ] 에러 배경색이 destructive 계열이 아닌 부드러운 톤이다
- [ ] 타임아웃 메시지에 파이뱀 선생님 스타일이 적용된다
- [ ] Pyodide 로드 실패 메시지가 파이뱀 선생님 스타일이다

### Task 8-4: 랜딩 페이지 파이뱀 선생님 환영 애니메이션

**문제**: 현재 정적 텍스트 + 버튼만 존재. PRD에서 "랜딩 페이지: 파이뱀 선생님 환영 애니메이션" 요구.

**수정 대상 파일**:
- `src/app/page.tsx` — 클라이언트 컴포넌트 분리
- (신규) `src/components/landing/HeroSection.tsx`

**구체적 변경**:
1. 파이뱀 선생님 이모지가 위에서 bounce-in으로 등장
2. "안녕! 나는 파이뱀 선생님이야~" 텍스트가 순차적으로 fade-in
3. "함께 파이썬 모험을 떠나볼래?" 텍스트 지연 등장
4. "Google로 시작하기" 버튼이 마지막에 scale + fade-in
5. Framer Motion `staggerChildren` + `variants` 패턴 사용

### Acceptance criteria
- [ ] 랜딩 페이지 진입 시 파이뱀 선생님 이모지가 bounce-in으로 등장한다
- [ ] 텍스트가 순차적으로 fade-in된다
- [ ] "Google로 시작하기" 버튼이 마지막에 나타난다

### Task 8-5: 월드맵 스테이지 등장/잠금 해제 애니메이션

**문제**: WorldMap에서 스테이지 노드가 정적으로 렌더링됨. 잠금 해제 시 시각적 효과 없음.

**수정 대상 파일**:
- `src/components/world/WorldMap.tsx` — stagger 진입 애니메이션
- `src/components/world/StageNode.tsx` — 잠금 해제 상태 강조 효과

**구체적 변경**:
1. WorldMap: 각 StageNode를 `motion.div`로 감싸고 `staggerChildren: 0.1`로 순차 등장 (fade-in + slide-up)
2. StageNode: `in_progress` 상태일 때 미세한 pulse glow 효과 (border 색상 애니메이션)
3. 연결선(세로 선분)도 순차적으로 그려지는 효과

### Acceptance criteria
- [ ] 월드맵 진입 시 스테이지 노드가 순차적으로 등장한다
- [ ] `in_progress` 상태 스테이지에 pulse glow 효과가 적용된다
- [ ] 연결선이 순차적으로 나타난다

### Task 8-6: 페이지 전환 애니메이션

**문제**: 라우트 이동 시 애니메이션 없음.

**수정 대상 파일**:
- (신규) `src/components/shared/PageTransition.tsx`
- `src/app/(protected)/world/page.tsx`
- `src/app/(protected)/profile/page.tsx`
- `src/app/(protected)/quest/[questId]/page.tsx`

**구체적 변경**:
- `PageTransition` 래퍼 컴포넌트: `motion.div` + `initial={{ opacity: 0, y: 12 }}` -> `animate={{ opacity: 1, y: 0 }}` + `transition={{ duration: 0.3, ease: 'easeOut' }}`
- 각 페이지의 `<main>` 또는 최상위 div를 `PageTransition`으로 래핑
- Next.js App Router에서는 `AnimatePresence` 기반 exit 애니메이션이 복잡하므로 진입 애니메이션만 적용

### Acceptance criteria
- [ ] `/world` 진입 시 fade+slide 애니메이션이 재생된다
- [ ] `/profile` 진입 시 fade+slide 애니메이션이 재생된다
- [ ] `/quest/[questId]`는 QuestShell이 자체 레이아웃이므로 별도 처리하지 않아도 됨

## 검증 방법

1. `npm run build` -- 빌드 에러 없음 확인
2. `npm run dev` -> Chrome DevTools Device Mode에서 360px / 768px / 1024px 확인
3. 각 페이지 진입 시 애니메이션 동작 확인 (랜딩, 월드맵, 퀘스트, 프로필)

5. OutputPanel에서 에러 발생 시 파이뱀 선생님 스타일 표시 확인
6. Pyodide 로딩 중 코딩 팁 로테이션 확인
7. 모든 클릭 타겟이 44px 이상인지 DevTools Inspector로 확인
