# Plan: pyRoad MVP 상세 개발 계획서

> Source PRD: `docs/2.PRD.md` (v1.0 MVP, 2026-03-30)

## Context

pyRoad는 AI 동물 마스코트 튜터("파이뱀 선생님")와 함께 퀘스트를 깨며 파이썬을 배우는 초등학생 전용 웹 학습 플랫폼이다. 현재 프로젝트에는 완성된 PRD(1122줄)만 존재하며, 소스 코드는 전혀 구현되지 않은 상태(`src/app/`, `src/lib/`에 `.gitkeep`만 존재). `package.json`, Next.js 설정, 의존성 등 아무것도 없다. 솔로 개발자 1인이 2~4주 내 MVP를 완성해야 한다.

## Architectural Decisions

모든 Phase에서 참조할 내구성 있는 결정 사항:

- **Routes**: `/` (랜딩/로그인), `/world` (월드맵, 보호), `/quest/[questId]` (퀘스트, 보호), `/profile` (프로필, 보호), `/api/tutor` (AI 튜터 API), `/auth/callback` (OAuth 콜백)
- **DB Schema**: Supabase PostgreSQL 5테이블 -- `users`, `stages`, `quests`, `user_progress`, `user_badges`. 모든 테이블 RLS 적용.
- **Auth**: Google OAuth 2.0 via Supabase Auth. Next.js 미들웨어 기반 라우트 보호.
- **LLM**: Anthropic Claude API, `/api/tutor` API Route에서 호출. 5개 프롬프트 유형(`quest_intro`, `hint_generator`, `code_feedback`, `encouragement`, `project_guide`), max 300 tokens, 실패 시 `prompt_skeleton` 폴백.
- **Code Execution**: Pyodide (브라우저 WASM, Web Worker), 5초 타임아웃, 허용 모듈(math, random, string), 차단 모듈(os, urllib, socket).
- **Validation**: `output_match`, `contains`, `code_check` 3종.
- **Gamification**: XP 힌트 보너스(0회: +50%, 1회: +25%, 2회+: 0%), 5레벨(0/100/300/600/1000 XP), 5뱃지(first_code, hint_master, stage_clear, project_builder, streak_3).
- **UI**: Tailwind CSS + shadcn/ui, Framer Motion + canvas-confetti, CodeMirror 6. 아동 친화 색상(녹색 Primary, 크림 Background). Pretendard/Noto Sans KR + JetBrains Mono.

## Phase 0: 프로젝트 부트스트랩 및 인프라

**User stories**: 없음 (기반 인프라)

### What to build

Next.js 15 프로젝트를 초기화하고 핵심 의존성(Tailwind CSS, shadcn/ui, Supabase SSR 클라이언트, Framer Motion, ESLint, Prettier, Vitest)을 설치한다. Supabase 프로젝트를 생성하고 5개 테이블 + RLS 정책 + 인덱스를 SQL 마이그레이션으로 적용한다. `.env.local` 및 `.env.example`을 작성한다. 루트 레이아웃에 글로벌 폰트(Pretendard/Noto Sans KR, JetBrains Mono)와 Tailwind 테마(아동 친화 색상 팔레트)를 설정한다. `/` 경로에 최소한의 페이지를 렌더링하여 `npm run dev`로 구동을 확인한다.

### Acceptance criteria

- [x] `npm run dev`로 로컬 서버가 정상 기동되고 `/`에서 페이지가 렌더링된다
- [x] `npm run build`가 에러 없이 성공한다
- [x] Supabase에 5개 테이블(`users`, `stages`, `quests`, `user_progress`, `user_badges`)이 생성되어 있다
- [x] 모든 테이블에 RLS 정책이 활성화되어 있다
- [x] `.env.example`에 필요한 환경 변수 키가 모두 명시되어 있다 (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`)
- [x] Tailwind CSS 테마에 아동 친화 색상 팔레트(Primary 녹색, Background 크림 등)가 설정되어 있다
- [x] ESLint, Prettier 설정이 적용되어 `npx eslint .`와 `npx prettier --check .`가 통과한다
- [x] Supabase SQL 마이그레이션 파일이 `supabase/` 디렉토리에 존재한다

## Phase 1: Google 로그인과 빈 월드맵

**User stories**: AUTH-01, AUTH-02, AUTH-03, AUTH-04, AUTH-05, MAP-01(부분)

### What to build

Google OAuth 로그인 전체 흐름을 구현한다. 랜딩 페이지(`/`)에 파이뱀 선생님 환영 텍스트와 "Google로 시작하기" 버튼을 배치한다. 버튼 클릭 시 Supabase Auth의 `signInWithOAuth`를 호출하고, `/auth/callback` 라우트에서 코드 교환을 처리한다. 첫 로그인 시 `users` 테이블에 레코드를 자동 생성한다(total_xp=0, current_level=1). Next.js 미들웨어에서 보호 경로(`/world`, `/quest/*`, `/profile`)에 대한 미인증 접근을 `/`로 리다이렉트한다. 로그인 성공 후 `/world`로 이동하며, 이 단계에서 월드맵은 "월드맵 준비 중" 플레이스홀더만 표시한다. `/profile` 경로에 사용자 이름, 아바타, 로그아웃 버튼을 배치한다.

### Acceptance criteria

- [x] Google 계정으로 로그인 시 `/world`로 리다이렉트된다
- [x] 첫 로그인 시 `users` 테이블에 레코드가 생성된다 (total_xp=0, current_level=1)
- [x] 브라우저 재방문 시 세션이 유지되어 자동 로그인된다
- [x] 미인증 상태에서 `/world`, `/quest/*`, `/profile` 접근 시 `/`로 리다이렉트된다
- [x] `/profile`에서 로그아웃 버튼 클릭 시 세션 종료 후 `/`로 이동한다
- [x] `/world`에 플레이스홀더 UI가 표시된다
- [x] Supabase Auth 콜백(`/auth/callback`)이 정상 동작한다

## Phase 2: 커리큘럼 시드 데이터와 월드맵

**User stories**: DATA-01, DATA-02, DATA-03, DATA-04, DATA-05, MAP-01, MAP-02, MAP-03, MAP-05

### What to build

7개 스테이지와 22개 퀘스트의 시드 데이터를 SQL INSERT 스크립트로 작성하여 Supabase에 삽입한다. 각 퀘스트에 `prompt_skeleton` JSON(topic, learning_goals, story_context, exercise_description, starter_code, expected_output_hint, fallback_text, hints)과 `validation_type` + `expected_output`을 포함한다. `/world` 페이지를 구현하여 7개 스테이지를 세로 경로(모험 지도) 형태로 시각화한다. 사용자의 `user_progress` 데이터를 조회하여 각 스테이지의 잠금/해제/진행중/완료 상태를 계산하고, 색상 + 아이콘으로 구분 표시한다. 해제된 스테이지 클릭 시 해당 스테이지의 첫 미완료 퀘스트(`/quest/[questId]`)로 내비게이션한다. 스테이지 1은 항상 해제 상태이다.

### Acceptance criteria

- [x] Supabase `stages` 테이블에 7개 스테이지 데이터가 존재한다
- [x] Supabase `quests` 테이블에 22개 퀘스트 데이터가 존재하며, 각각 유효한 `prompt_skeleton`과 `validation_type`/`expected_output`을 포함한다
- [x] SQL 시드 스크립트가 `supabase/seed.sql`로 제공되며 Supabase SQL Editor에서 실행 가능하다
- [x] `/world`에서 7개 스테이지가 모험 지도 형태로 표시된다
- [x] 스테이지 1은 항상 해제(in_progress) 상태이며, 나머지는 이전 스테이지 완료 전까지 잠금 표시된다
- [x] 각 스테이지에 완료/진행중/잠금 상태가 시각적으로 구분된다 (색상 + 아이콘)
- [x] 해제된 스테이지 클릭 시 해당 스테이지의 첫 미완료 퀘스트 `/quest/[questId]`로 이동한다
- [x] 잠금 상태 스테이지 클릭 시 이동하지 않는다

## Phase 3: 퀘스트 화면 골격과 Pyodide 코드 실행

**User stories**: QUEST-01, QUEST-02, QUEST-04, QUEST-05, QUEST-06, EXEC-01, EXEC-02, EXEC-03, EXEC-04, EXEC-05, EXEC-06, EXEC-07, EXEC-08

### What to build

`/quest/[questId]` 페이지의 Split View 레이아웃을 구현한다. 데스크톱(>=768px)에서는 좌측에 대화 패널(이 단계에서는 `prompt_skeleton.fallback_text`를 정적 표시), 우측에 CodeMirror 6 에디터 + 실행 결과 패널을 배치한다. 모바일(<768px)에서는 대화/코드/결과를 탭으로 전환한다. Pyodide Web Worker를 구현하여 "코드 돌려보기!" 버튼 클릭 시 Python 코드를 브라우저 내에서 실행한다. stdout/stderr를 캡처하여 결과 패널에 표시한다. 5초 타임아웃, 허용/차단 모듈 제한을 적용한다. Pyodide WASM은 퀘스트 페이지 진입 시 Lazy Loading하며, 로딩 중 안내 UI를 표시한다. 퀘스트 데이터를 Supabase에서 조회하여 에디터에 `starter_code`를 초기 세팅한다.

### Acceptance criteria

- [x] `/quest/[questId]`에서 퀘스트 데이터가 Supabase에서 로딩되어 표시된다
- [x] 데스크톱(>=768px)에서 Split View(좌: 대화, 우: 에디터+결과)가 표시된다
- [x] 모바일(<768px)에서 대화/코드/결과 탭 전환 방식으로 동작한다
- [x] CodeMirror 6 에디터에서 Python 구문 하이라이팅이 적용된다
- [x] "코드 돌려보기!" 버튼 클릭 시 Pyodide가 코드를 실행하고 stdout이 결과 패널에 표시된다
- [x] stderr(에러)가 결과 패널에 친근한 UI로 표시된다
- [x] 5초 초과 실행 시 "코드가 너무 오래 걸려요!" 메시지가 표시된다
- [x] `os`, `urllib`, `socket` 등 차단 모듈 import 시 적절한 안내가 표시된다
- [x] Pyodide 로딩 중 로딩 UI가 표시된다
- [x] 퀘스트의 `starter_code`가 에디터에 초기 세팅된다
- [x] 상단 헤더에 "← 월드맵" 뒤로가기 링크와 현재 스테이지/퀘스트 정보가 표시된다

## Phase 4: AI 튜터 API와 퀘스트 도입/힌트

**User stories**: AI-01, AI-02, AI-05, AI-06, AI-07, AI-08, QUEST-03, QUEST-07

### What to build

`/api/tutor` API Route를 구현한다. Supabase Auth JWT를 검증한 후, 요청 유형(`quest_intro`, `hint_generator`)에 따라 시스템 프롬프트(파이뱀 선생님 페르소나 + 안전 지침) + 커리큘럼 컨텍스트(`prompt_skeleton`)를 조합하여 Anthropic Claude API를 호출한다. max_tokens=300을 적용한다. API 실패 시 1차 재시도(1초 후), 재실패 시 `prompt_skeleton`의 `fallback_text` 또는 정적 힌트(`hints.level_N`)로 폴백한다. 퀘스트 진입 시 `quest_intro` 프롬프트로 도입 스토리를 생성하여 대화 패널에 표시한다. "힌트" 버튼 클릭 시 현재 힌트 레벨(1/2/3)에 따라 에스컬레이션 힌트를 생성한다. LLM 응답 대기 중 파이뱀 선생님 생각 애니메이션을 표시한다.

### Acceptance criteria

- [x] `/api/tutor`가 Supabase Auth JWT를 검증하고, 미인증 요청을 401로 거부한다
- [x] `quest_intro` 요청 시 파이뱀 선생님이 스토리 기반 도입 메시지를 생성한다
- [x] `hint_generator` 요청 시 힌트 레벨(1/2/3)에 따라 점진적으로 구체적인 힌트를 생성한다
- [x] LLM 응답이 300토큰 이내로 제한된다
- [x] LLM API 실패 시 1차 재시도 후, 재실패 시 `prompt_skeleton`의 폴백 텍스트가 표시된다
- [x] LLM 응답 대기 중 파이뱀 선생님 생각 애니메이션이 표시된다
- [x] 퀘스트 진입 시 자동으로 도입 스토리가 대화 패널에 표시된다
- [x] "힌트" 버튼이 최대 3회까지 동작하며, 힌트 사용 횟수가 UI에 표시된다 (예: "힌트: 1/3 사용")
- [x] 학습 범위 외 질문에 대해 시스템 프롬프트 가드레일이 적용된다

## Phase 5: 코드 검증과 퀘스트 완료 처리

**User stories**: QUEST-08, QUEST-09, QUEST-10, AI-03, AI-04, GAME-01, GAME-02

### What to build

코드 실행 후 `validation_type`(`output_match`, `contains`, `code_check`)에 따른 자동 검증 로직을 구현한다. 검증 성공 시: XP를 계산하고(기본 XP + 힌트 보너스: 0회 +50%, 1회 +25%, 2회+ +0%) `users.total_xp`를 업데이트하며, `user_progress`에 status='completed', hints_used, code_submitted, completed_at을 저장한다. `/api/tutor`에 `code_feedback` 프롬프트 유형을 추가하여 검증 결과(성공/실패/에러)에 대한 AI 피드백을 생성한다. `encouragement` 프롬프트 유형을 추가하여 퀘스트 완료 시 축하 메시지를 생성한다. 검증 실패 시 정답을 알려주지 않고 힌트를 제공한다. 퀘스트 진행 중 코드와 힌트 사용 횟수를 `user_progress`에 자동 저장한다.

### Acceptance criteria

- [x] `output_match` 검증: stdout이 `expected_output`과 정확히 일치할 때 통과한다
- [x] `contains` 검증: stdout에 `expected_output`이 포함되어 있을 때 통과한다
- [x] `code_check` 검증: 학생 코드에 `expected_output` 문자열이 포함되어 있을 때 통과한다
- [x] 검증 성공 시 XP가 계산되어 `users.total_xp`에 반영된다 (힌트 0회: +50%, 1회: +25%, 2회+: +0%)
- [x] 검증 성공 시 `user_progress`에 완료 상태(status='completed')가 저장된다
- [x] `code_feedback` 프롬프트로 실행 결과에 대한 AI 피드백이 대화 패널에 표시된다
- [x] `encouragement` 프롬프트로 완료 시 축하 메시지가 대화 패널에 표시된다
- [x] 검증 실패 시 "앗, 조금 고쳐볼까요?" 안내와 함께 힌트 사용을 유도한다
- [x] 퀘스트 재진입 시 이전에 저장한 코드가 에디터에 복원된다
- [x] 하단 상태바에 현재 XP, 레벨, 힌트 사용 횟수가 표시된다

## Phase 6: 게이미피케이션 (레벨, 뱃지, 축하 애니메이션)

**User stories**: GAME-03, GAME-04, GAME-05, GAME-06, GAME-07, MAP-04, MAP-06

### What to build

레벨 시스템을 구현한다. XP 획득 시 누적 XP가 레벨 임계값(100/300/600/1000)을 넘으면 `users.current_level`을 업데이트하고 레벨업 알림(축하 애니메이션 + 새 칭호)을 표시한다. 뱃지 시스템을 구현한다. 퀘스트 완료 시 5종 뱃지 조건을 체크하여 조건 충족 시 `user_badges`에 삽입하고 팝업 알림을 표시한다:
- `first_code`: 첫 번째 퀘스트 완료
- `hint_master`: 힌트 0회로 퀘스트 완료
- `stage_clear`: 한 스테이지의 모든 퀘스트 완료
- `streak_3`: 3일 연속 퀘스트 완료

축하 애니메이션에 canvas-confetti와 Framer Motion을 사용한다. `/world` 월드맵 상단에 현재 레벨, 칭호, 총 XP, 아바타, 뱃지 컬렉션(획득 아이콘 + 미획득 실루엣)을 표시한다. `/profile` 페이지에 진행도 시각화(스테이지별 퍼센트, XP 바, 뱃지 목록, 완료 퀘스트 수/전체)를 구현한다.

### Acceptance criteria

- [x] 누적 XP가 임계값(100/300/600/1000)을 넘으면 레벨이 자동으로 올라간다
- [x] 레벨업 시 confetti + 새 칭호 표시 축하 애니메이션이 재생된다
- [x] 첫 퀘스트 완료 시 `first_code` 뱃지가 획득된다

## Phase 7: 최종 프로젝트 가이드 (스테이지 7)

**User stories**: PROJ-01, PROJ-02, PROJ-03, PROJ-04, PROJ-05

### What to build

스테이지 7 "프로젝트 왕국"의 가이드 빌딩 방식 프로젝트를 구현한다. 프로젝트("숫자 맞추기 게임")를 5~6단계로 나누어 `/api/tutor`에 `project_guide` 프롬프트 유형을 추가한다. 각 단계에서 LLM이 이전 단계 코드를 참조하여 다음 기능 추가를 안내하고, 학생이 코드를 작성하면 단계별 검증을 수행한 후 다음 단계로 진행한다. 프로젝트 완료 시 특별 축하 애니메이션(강화된 confetti + 파이뱀 선생님 춤)과 `project_builder` 뱃지를 부여한다. 프로젝트 퀘스트의 멀티스텝 시드 데이터(`prompt_skeleton`에 단계별 step_goal 포함)를 추가한다.

### Acceptance criteria

- [x] 스테이지 7 진입 시 프로젝트 가이드 빌딩 흐름이 시작된다
- [x] `project_guide` 프롬프트로 LLM이 각 단계의 목표를 안내한다
- [x] LLM이 이전 단계에서 작성한 코드를 참조하여 안내한다
- [x] 각 단계에서 코드 검증을 통과해야 다음 단계로 진행할 수 있다
- [x] 5~6단계를 순차적으로 완료할 수 있다
- [x] 프로젝트 완료 시 특별 축하 애니메이션이 재생된다
- [x] `project_builder` 뱃지가 자동 부여된다
- [x] 프로젝트 진행 중 단계별 코드가 자동 저장된다

## Phase 8: 반응형 UI 폴리싱 및 애니메이션

**User stories**: 6.2 UX 가이드라인, 6.3 반응형 디자인, 6.4 애니메이션 요구사항

### What to build

전체 페이지의 반응형 디자인을 360px~1024px+ 범위에서 검수하고 보정한다. 아동 친화 UX 가이드라인을 적용한다:
- 클릭 타겟 최소 44px
- 버튼 클릭 시 Framer Motion scale/spring 효과
- 에러 메시지에 빨간 경고 대신 파이뱀 선생님 안내 적용
- 모든 UI 텍스트를 쉬운 한국어 용어로 통일 ("코드 돌려보기!", "앗, 조금 고쳐볼까요?", "대단해요!")

애니메이션 구현:
- 랜딩 페이지: 파이뱀 선생님 환영 애니메이션
- 퀘스트 완료: 파이뱀 선생님 춤/기쁨 + XP 카운트업
- 스테이지 잠금 해제: 잠금 해제 애니메이션
- 페이지 전환: 라우트 이동 애니메이션
- Pyodide 로딩: 코딩 팁 + 준비 애니메이션

### Acceptance criteria

- [ ] Chrome DevTools에서 360px, 768px, 1024px 해상도에서 모든 페이지가 정상 표시된다
- [ ] 모든 클릭 타겟이 최소 44px이다
- [ ] 버튼 클릭 시 scale/spring 애니메이션이 적용된다
- [ ] 에러 상황에서 빨간 경고 대신 파이뱀 선생님 안내가 표시된다
- [ ] 모든 UI 텍스트가 아동 친화 용어로 통일되어 있다
- [ ] 랜딩 페이지에 파이뱀 선생님 환영 애니메이션이 재생된다
- [ ] 퀘스트 완료 시 XP 카운트업 애니메이션이 재생된다
- [ ] 스테이지 잠금 해제 시 해제 애니메이션이 재생된다
- [ ] 페이지 전환 시 애니메이션이 적용된다
- [ ] Pyodide 로딩 중 코딩 팁이 표시된다

## Phase 9: 통합 테스트 및 배포

**User stories**: PRD 섹션 10 성공 지표 전체

### What to build

MVP 완료 기준(PRD 섹션 10.1) 10개 항목을 순차 검증한다. Vitest로 핵심 유틸리티(XP 계산, 검증 로직, 레벨/뱃지 판정, 스테이지 잠금/해제 로직) 단위 테스트를 작성한다. Playwright로 주요 E2E 시나리오를 자동화한다:
1. 로그인 -> 월드맵 -> 퀘스트 진입 -> 코드 실행 -> 검증 성공 -> XP 획득 -> 월드맵 복귀
2. 힌트 3단계 에스컬레이션
3. LLM 폴백 동작 (API 키 제거 후 퀘스트 진입 시 정적 텍스트 표시)

Vercel 배포 설정을 구성하고, 환경 변수를 Vercel에 등록하여 프로덕션 배포를 수행한다. Lighthouse 성능 검수를 실행한다.

### Acceptance criteria

- [ ] Vitest 단위 테스트가 모두 통과한다 (XP 계산, 검증 로직, 레벨/뱃지 판정)
- [ ] Playwright E2E 테스트가 주요 시나리오를 통과한다
- [ ] LLM 폴백이 정상 동작한다 (API 키 없이도 퀘스트 진행 가능)
- [ ] Vercel에 프로덕션 배포가 성공적으로 완료된다
- [ ] Lighthouse Performance >= 80
- [ ] Lighthouse Accessibility >= 90
- [ ] PRD 섹션 10.1의 10개 MVP 완료 기준을 모두 충족한다:
  - [ ] Google 로그인 가능
  - [ ] 7개 스테이지 + 22개 퀘스트 시딩 완료
  - [ ] 월드맵 잠금/해제 정상 동작
  - [ ] AI 튜터 5개 유형 프롬프트 동작
  - [ ] Pyodide 코드 실행 + 결과 표시
  - [ ] 3종 코드 검증 동작
  - [ ] XP/레벨/뱃지 시스템 동작
  - [ ] 최종 프로젝트 완주 가능
  - [ ] 360px~데스크톱 반응형 정상
  - [ ] LLM 폴백 정상 동작
